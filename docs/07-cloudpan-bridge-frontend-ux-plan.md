# CloudPan Bridge 控制台前端重构计划（易上手版）

## 目标

把当前“功能都能用，但页面层级太平、普通人第一次进来容易迷路”的控制台，重构成更容易理解的主流程界面：

- 顶层只保留主流程 Tab
- 高级项、诊断项、帮助项下沉到二级弹窗
- 执行增强能力改成折叠区，而不是和主操作并排抢注意力
- 普通用户第一次进入后，能顺着“连接 -> 源端 -> 目标端 -> 任务 -> 执行”快速完成一次闭环

## 信息架构

- 顶级导航固定为 6 个：
  - 总览
  - 连接
  - 源端
  - 目标端
  - 任务
  - 执行
- 不再保留顶层独立页：
  - 补传
  - 秒传
  - 关于
- 新增二级入口：
  - 帮助与能力中心弹窗
  - 高级设置弹窗
  - 驱动接入流程弹窗
  - 日志抽屉

## 页面收口原则

- 总览：
  - 只放状态摘要、推荐下一步、快捷跳转
- 连接：
  - 默认只显示当前 OpenList 模式
  - 挂载总览、网页登录抓取、手动新增挂载默认收起
- 源端：
  - 默认只保留目录浏览与 source_path 选择
  - 源端分析与覆盖项默认收起
- 目标端：
  - 默认只保留目标端选择、目标端凭证、目标挂载选择
  - 日志/状态/临时目录/速率页数等移出主区，放入高级设置
- 任务：
  - 只保留 source_path、target_key、target_path、同步策略
  - OpenList 目标挂载继续双入口保留
- 执行：
  - 第一屏只保留 dry-run / 直接同步 / 队列执行
  - 补传、秒传、能力矩阵改成折叠增强区

## 第一阶段实施项

- 移除顶层 `补传 / 秒传 / 关于` Tab
- 在执行页内合并：
  - 待补传目录树
  - 秒传 JSON 直导
  - 能力矩阵与建议
- 新增“帮助与能力中心”弹窗，承接原关于页内容
- 新增“高级设置”弹窗，承接：
  - 控制台管理员
  - 日志/临时目录/状态文件
  - OpenList 页数与间隔配置
- 把连接页中的：
  - OpenList 挂载总览
  - 源网盘登录抓取
  - 新增挂载
  改成默认收起

## 验收标准

- 页面首次进入时，顶层导航不再超过 6 个
- 普通用户不需要打开关于页也能完成一次主流程
- 执行页第一屏不再出现大量低频说明块
- 帮助、能力矩阵、驱动覆盖审计仍可访问，但不抢主流程注意力
- 现有功能不丢：
  - OpenList 登录写回
  - 目录浏览
  - 目标端切换
  - OpenList 目标挂载双入口
  - 队列执行
  - 补传勾选树
  - 秒传 JSON 导入
